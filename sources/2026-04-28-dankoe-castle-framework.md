# Source: Dan Koe — CASTLE Framework

Xiaohongshu post presenting Dan Koe's "CASTLE 5步法" (5-step method) for opening your first business/validating a first startup idea.

## Key content

| Letter | Step | Description |
|---|---|---|
| C | Crystallize Direction | Start with problems you personally suffered through, tried, and overcame — lived experience is your most credible starting point |
| A | Architecture Path | Don't rush into complex business theory; observe how others explain similar problems; extract a simple framework: pain points → transformation → steps → examples → actions |
| S | Share Openly | Document immediately, before you're an expert — record learning, experiments, obstacles, and progress consistently |
| T | Test Lightly | Package methodology into the smallest deliverable unit: checklist, template, manual, or single conversation. Helpfulness over perfection |
| L | Listen to Feedback | Use the first 10 users to refine approach and uncover blind spots — genuine responses, not validation of expertise |

Core philosophy: don't try to "do big" — select one problem, publicly solve it, package the solution for others. Creates the cycle: learn → practice → receive feedback → improve.

- The post explicitly says "5步法" (5-step method) but CASTLE has 6 letters. The **"E" step is absent** from this summary — either the Xiaohongshu author adapted the acronym for 5 steps or the fetch missed it.
